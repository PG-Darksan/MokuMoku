功能介绍

— Windows 快捷键,不离开键盘飞速操作。

— 01 / Editing & creation

编辑和创建节点

将选中节点进入编辑模式、添加新节点、扩展分支、折叠子孙 — 使用频率最高的快捷键。

按键 动作 概述
F2 编辑节点 立即编辑选中节点。按 F2 一次即可进入文本编辑,无需双击。
Ctrl+Shift+N 创建节点 添加一个新节点。如果有选中节点,旁边会出现空节点。开始绘图时第一个按下的键。
Ctrl+Shift+A 批量创建子节点 选中单个节点后,可批量创建指定数量的空子节点。无需 AI 也能快速扩展。
Ctrl+Shift+W 创建共同父节点 范围选择多个节点后,创建一个公共父节点将它们作为子节点包裹。
Ctrl+Q 折叠/展开子节点 选择有子节点的父节点,隐藏其下子孙(收纳)。再按展开。仅用左手快速操作。
DEMO — F2 / Ctrl+Shift+N / Ctrl+Q 组合实现节点编辑、创建、折叠
— 02 / Grouping

分组节点

范围选择した複数ノードを 1 つにまとめたり、 付箋でゆるく束ねたり、 線をなぞって接続を一気に切ったり 大規模マップの構造を 自在に組み替える為の按键群です

按键 动作 概述
Shift+Drag 范围选择 按住 Shift 拖动,选择矩形内所有节点。然后可分组、移动、改色。
Ctrl+G 合并为一个节点 范围选择したノード群を 合并为一个节点 。原节点被捆绑作为单一节点处理。用于大胆简化。
Ctrl+Shift+G 使用便签分组 范围选择したノード群に 付箋名 附加便签名松散捆绑。保留各节点,可一起拖动或改色。に
Ctrl+Shift+V 切换裁断模式 切换到描线即可切断连接的模式。再按返回。无需逐条右键删除。
DEMO — Shift+Drag 范围选择 / Ctrl+G 集約 / Ctrl+Shift+G 付箋 / Ctrl+Shift+V 裁断
— 03 / Cloud sync

云同步

同步到 Firebase、重新获取、管理同步组。每个工作节点一键安心。

按键 动作 概述
Ctrl+S 云同步(上传) 将当前地图同步到 Firebase,反映本地编辑。按 Enter 确认对话框默认选项。
Ctrl+D 云端下载(重取) 从云端重新获取地图。用于拉取其他设备的编辑。日历模式下变为获取日历事件。
Ctrl+Shift+S 打开同步组 列出同步对象所有思维导图 同步组 面板。一目了然哪些已同步、哪些未同步。
DEMO — Ctrl+S アッ专业ード / Ctrl+D 取り直し / Ctrl+Shift+S 同步组
— 04 / View switching

视图切换

调用侧边菜单、瞬间切换思维导图、管理切换时中心位置。多地图工作者的快捷键。

按键 动作 概述
Ctrl+Shift+E 打开侧边菜单 开关侧边菜单访问思维导图列表、文件夹、设置。无需移动鼠标到屏幕边缘。
Ctrl+1 〜 Ctrl+9 跳转到第 1-9 个地图 瞬间跳到抽屉中第 1-9 个地图。常用地图放顶部,一键切换。
Ctrl+B 设置基准点 设置切换时的 基准点 为当前视图位置。记住"这张图从这里查看"的基点。
Ctrl+Shift+B 前往基准点 Ctrl+B で覚えさせた基准点へ、 ビューを一気に戻します マップ内を 探索した後でも、 1 押しで「最初に決めた中心」 に帰還できます
DEMO — Ctrl+Shift+E サイドメニュー / Ctrl+1〜9 マップ移動 / Ctrl+B 基准点 / Ctrl+Shift+B 帰還
— 05 / Other features

其他功能

在绘图时计算、计时、看天气。直达内置工具,无需切换应用。

按键 动作 概述
Ctrl+Shift+T 亮色/暗色切换 一键反转主题。夜晚暗色,白天亮色。减轻眼睛负担。
Ctrl+Shift+C 打开日历 弹出内置日历。点击日期看当天日程和地图关联事件。
Ctrl+M 打开计算器 立即调用标准计算器。四则运算和 % 等,快速计算。
Ctrl+Shift+M 高度打开计算器 科学计算器模式。三角函数、对数、幂、括号等复杂计算。
Ctrl+W 打开天气 基于位置显示当地天气。无需切换标签即可查看。
Ctrl+T 秒表/计时器 启动计时或倒计时。记录工作时间、设置专注时段。
Ctrl+P 番茄钟 专用番茄钟:25 分钟工作 + 5 分钟休息。长时间专注工作的伙伴。
DEMO — Ctrl+Shift+T 主题 / Ctrl+M 计算器 / Ctrl+W 天气 / Ctrl+T 秒表 / Ctrl+P 番茄钟

所有快捷键均可重新分配。

在应用内, 设置 → 快捷键 可更改每个命令的键。冲突自动检测。Windows 标准复制/粘贴/撤销在文本框中遵循系统默认。快捷键请求通过 联系我们 直接发送给开发者。
